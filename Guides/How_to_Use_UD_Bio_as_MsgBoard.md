# How to Use a UD Bio as a WEB3MSGBOARD Node

**WEB3MSGBOARD — a Distributed Status Mesh**

> WEB3MSGBOARD is a distributed check-in system where each person controls their own writable status page and others know where to find it when systems fail.

---

## Important Shift in Model

This is **not** a shared group message board.

This is a **distributed mesh**:

- Each person owns **their own** domain
- Each person writes **their own** status
- Others can **view** it
- No one waits for permission
- No central admin
- No bottleneck

This is intentional.

---

## What the UD Bio Is Used For

The Unstoppable Domains Bio section functions as a:

✔ Viewable status bulletin  
✔ Simple fallback page  
✔ Public check-in surface  
✔ Low-bandwidth update point  

It is **not** a chat, feed, or comment wall.

---
